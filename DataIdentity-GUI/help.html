Project Collaboration

Globally unique Persistent IDs (PIDs) promise to make research data easier to manage. The RDA Recommendations, Persistent Identifier Types (PIT) and Data Type Registry (DTR), are enabling solutions. A group of us in the Pacific Rim Applications and Grid Middleware Assembly (PRAGMA), a strong network of member institutions around the Pacific Rim, adopted PIT and DTR into a application that runs Galaxy workflows on PRAGMA VMs running on the PRAGMA Cloud Testbed.



RDA PIT/DTR Service

Identity and interoperability of PIDs resolved among different systems and providers is needed. RDA PIT working group allows providers agree on a common API, register their information types in a common data type registry and agree on some core types, such as EUDAT profile with useful metadata units.

Service Deployment

In this project, we provide a framework with both repository and PID service (PRAGMA Data Identity Service and PRAGMA Data Repository) to provide long-term access and findability to heterogeneous data objects (DOs) across scientific domains.

Application

  • Rice genomics Tassel5 workflow, a Galaxy based interactive workflow that enables collaborators to do their own GWAS analysis of their own phenotyping data with 3K Rice Genomes using common analysis framework
  • IRRI is willing to provide analysis framework for free, but wants researchers to share results back to IRRI. IRRI is interested in reproducibility of results as well

Deployment

  • Deployed PRAGMA data identity service, data repository and RDA PIT/DTR service on one VM cluster in the National Institute of Advanced Industrial Science and Technology (AIST), Tsukuba, Japan;
  • On Rice Genomics Workflow VM in SDSC, US, we set up Galaxy workflow server with Tassel5 workflow and data identity service client and GUI;
  • CNRI in Virginia, US provides one Handle V8 instance with a persistent Handle prefix for our evaluation.
